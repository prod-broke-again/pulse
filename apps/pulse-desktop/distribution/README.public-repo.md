# Pulse Desktop — дистрибуция

Этот файл можно скопировать в корень **отдельного публичного** репозитория на GitHub: там не хранится исходный код — только инструкция для пользователей и **релизы** с установщиками. Приватный репозиторий с кодом не светится.

## Скачать

1. Откройте вкладку [Releases](https://github.com/prod-broke-again/pulse-desktop-releases/releases) этого репозитория.
2. Выберите последний релиз и скачайте установщик под вашу ОС (Windows — `Pulse-Windows-*-Setup.exe` и т.д.).

Обновления доставляются через встроенный механизм приложения (GitHub Releases). После установки новых версий достаточно согласиться на обновление, когда приложение предложит.

## Безопасность

- Мы не публикуем исходники здесь; только проверяемые бинарники в Releases.
- Проверяйте, что релиз подписан вашим процессом релиза и совпадает с тегом в Git.

## Для сопровождения (команда разработки)

Репозиторий создаётся пустым (или только с этим README). Билд делается **в приватном** репозитории с кодом.

### 1. Одноразовая настройка

1. Создайте публичный репозиторий, например `pulse-desktop-releases`.
2. В `apps/pulse-desktop/electron-builder.json5` в приватном репозитории укажите тот же `publish.owner` и `publish.repo`, что и у публичного репо.
3. Соберите установщик: из каталога `apps/pulse-desktop` выполните `npm install` и `npm run build`.

### 2. Публикация релиза

После сборки артефакты лежат в `apps/pulse-desktop/release/<версия>/` (например `release/1.0.4/`).

1. В GitHub публичного репозитория: **Releases → Draft a new release**.
2. **Tag** — тег вида `v1.0.4` (как минимум должен совпадать с версией в `package.json`).
3. Загрузите **все** файлы, которые сгенерировал electron-builder для этой платформы, в том числе:
   - установщик (`.exe` / `.dmg` / `.AppImage`);
   - `latest.yml` (Windows), `latest-mac.yml` (macOS), `latest-linux.yml` (Linux) — **без них автообновление не найдёт новую версию**.
4. Опубликуйте релиз (не оставляйте черновиком).

### 3. Альтернатива: публикация из CLI приватного репо

Если настроен `GH_TOKEN` с правом **contents** на публичный репозиторий, можно использовать `electron-builder` с опцией публикации в GitHub (см. документацию electron-builder `publish`). Обычно достаточно ручной загрузки артефактов — проще контролировать, что именно уходит пользователям.

### Замечания

- Теги и номер версии в `package.json` должны быть согласованы.
- В `electron-builder.json5` для прод используются `publish.owner`: `prod-broke-again`, `publish.repo`: `pulse-desktop-releases`.
