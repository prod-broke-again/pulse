# Pulse Design Prompts (Desktop + Mobile)

Ниже готовые промпты для нейросети-дизайнера.  
Сначала идет универсальный контекст проекта, затем 3 отдельных промпта:
1) Desktop, 2) Mobile Flutter, 3) Mobile Capacitor + Vue + Tailwind + DaisyUI.

---

## 1) Универсальный контекст проекта (вставлять в начало любого промпта)

Ты проектируешь интерфейс для продукта **Pulse** — омниканального центра обработки обращений пользователей.  
Проект делается для **Портала АЧПП**:

**Ассоциация частнопрактикующих психологов и психотерапевтов**  
Пространство, объединяющее психологов и психотерапевтов, реализующих свои услуги в формате частной практики.  
Мы создаем поддерживающую среду для специалистов, выбравших самостоятельный путь, и помогаем выстраивать устойчивую, гармоничную и этичную частную практику.

### Суть продукта
- Pulse собирает сообщения из разных каналов (Telegram, VK, web-виджет) в одном рабочем месте.
- Модераторы должны быстро обрабатывать вопросы и жалобы без переключения между платформами.
- Важны скорость, аккуратность коммуникации, эмпатия и прозрачность статусов.

### Основные роли
- **Модератор**: основной пользователь интерфейса (входящие, ответы, эскалация, закрытие диалогов).
- **Администратор**: управляет доступами, источниками, настройками.

### Ключевые сущности интерфейса
- **Chat**: диалог с пользователем, статус (`open/closed`), назначение на модератора.
- **Message**: сообщение клиента/модератора/системы, текст и вложения.
- **Source**: канал обращения (VK, Telegram, Web).
- **Department**: подразделение/направление маршрутизации.
- **Canned Responses**: шаблоны быстрых ответов.

### AI-функции (обязательно учесть в UX)
- AI помогает модератору формулировать ответ.
- AI предлагает готовые варианты решения по контексту переписки и данным пользователя.
- AI может кратко суммировать суть обращения (тема/интент), чтобы ускорить triage.
- AI-подсказки не заменяют модератора, а усиливают его (human-in-the-loop).

### UX-приоритеты
- Минимум когнитивной нагрузки при большом потоке чатов.
- Быстрый triage: фильтры, приоритеты, назначение, статусы.
- Отличная читаемость длинных переписок.
- Явные состояния: loading, empty, error, offline, no results.
- Доступность (контраст, крупные кликабельные зоны, понятная типографика).
- Спокойный визуальный тон: доверие, поддержка, профессионализм, без тревожной агрессии.

### Бренд-палитра (обязательно использовать)

### Правила dark theme
- Основа dark-темы: `zinc`, но не «угольно-черная» (не перегружать `#000` фоном).
- Рекомендуемый диапазон фонов: `zinc-900` / `zinc-950` c мягкими слоями и контрастными карточками.
- Брендовый фиолетовый использовать как аккуратный акцент на CTA, active states, focus rings.

### Визуальный референс по стилистике
- Ориентир по mood и контрастам: как в связанном каталоге АЧПП/«Аврора» — чистые поверхности, `zinc` в темной теме, фиолетовые бренд-акценты, мягкие границы, спокойная типографика.
- Не копируй каталог буквально; перенеси только визуальные принципы в интерфейс модерации чатов.

## 2) Промпт для Desktop-дизайна (основной рабочий инструмент модератора)

Сгенерируй **дизайн desktop-приложения** для продукта Pulse (support cockpit для модераторов), с упором на высокую скорость обработки обращений.

Используй этот контекст проекта:
- Pulse объединяет чаты из Telegram/VK/Web в единый inbox.
- Целевая аудитория интерфейса: модераторы Портала АЧПП.
- Нужна спокойная, профессиональная, эмпатичная визуальная атмосфера.
- Основная задача: удобно отвечать на вопросы и жалобы, не теряя контекст.

### Что спроектировать
- Полноценный desktop layout (min width ~1180+), 3-зонная структура:
  1) узкая навигационная колонка,
  2) список чатов (inbox),
  3) активная переписка (thread + composer).
- Верхняя панель окна с кастомными оконными действиями (minimize/maximize/close).
- Список чатов с табами: `Мои`, `Нераспределенные`, `Все`.
- Фильтры: `Открытые/Закрытые`, поиск, индикаторы приоритета/непрочитанного.
- Карточка чата: канал (VK/TG/Web), имя/ID пользователя, краткий превью-текст, время, статус.
- Экран переписки: хедер, лента сообщений, вложения, действие `Назначить на меня`, `Закрыть чат`.
- Composer: поле ответа, прикрепление файлов, доступ к шаблонам ответов.
- AI-панель (встроенная в thread или composer):  
  - “Сгенерировать черновик ответа”,  
  - “Предложить 3 варианта ответа (эмпатичный/нейтральный/формальный)”,  
  - “Краткое резюме проблемы”,  
  - “Рекомендованное решение”.

### Обязательные состояния
- Loading skeletons (для inbox и thread).
- Empty states (нет чатов, не выбран чат, нет сообщений).
- Error states (с retry).
- Long conversation mode (удобная вертикальная сканируемость).
- Sticky composer + сохранение черновика.

### Визуальная система
- Modern calm UI (glassmorphism/light elevation допустимы, но без перегруза).
- Аккуратная типографика, контрастные тексты, мягкие акценты.
- Цветовая логика:
  - нейтральный фон на базе `--color-light` и бренда,
  - акцент для primary actions на базе `--color-brand`/`--color-brand-hover`,
  - отдельные маркеры для urgent/unread/warning.
- В dark-теме используй `zinc`-подложку средней темноты (не слишком темную) + брендовые акценты.
- Компоненты должны быть реалистично implementable.

### Вывод
Дай результат как:
1. Design direction (2-3 абзаца).  
2. Layout breakdown по блокам.  
3. Компонентная спецификация (sidebar, inbox item, thread, composer, AI card).  
4. State matrix (loading/empty/error/success).  
5. Токены (color, typography, spacing, radius, shadow).  
6. Короткий UX rationale: почему решение ускоряет модераторов.

## 3) Промпт для Mobile Flutter (нативный UX)

Сгенерируй **mobile app UX/UI-концепт под Flutter** для продукта Pulse (модерация входящих обращений).

Контекст:
- Pulse собирает чаты из Telegram/VK/Web в одном месте.
- Пользователи приложения: модераторы Портала АЧПП, работающие “на ходу”.
- Цель: быстро просматривать входящие, отвечать, назначать/закрывать чаты, использовать AI-помощника.

### Ограничения мобильного сценария
- One-hand use, быстрые действия большим пальцем.
- Максимально короткие пользовательские пути (few taps).
- Отличная читаемость текста в длинных диалогах.
- Должно работать в light/dark.
- Light/dark должны опираться на бренд-палитру из общего контекста; dark на базе `zinc` и без излишне темного фона.

### Что спроектировать (Flutter-first)
- Основная навигация: bottom navigation (например: `Inbox`, `Active`, `AI`, `Settings`) или другой нативный паттерн с обоснованием.
- Экран списка чатов:
  - сегменты `Мои/Нераспределенные/Все`,
  - поиск,
  - фильтр `Открытые/Закрытые`,
  - карточки с каналом, превью, временем, индикатором непрочитанного/срочности.
- Экран диалога:
  - хедер с данными чата и действиями (`Назначить`, `Закрыть`),
  - лента сообщений,
  - вложения,
  - composer с быстрым доступом к шаблонам.
- AI assistant sheet/bottom panel:
  - “Суммаризируй диалог”,
  - “Предложи ответ”,
  - “Предложи план решения”.
- Push/notification UX:
  - сценарий открытия конкретного чата по пушу.

### Состояния и доступность
- Loading, empty, no network, error with retry.
- Большие touch targets, понятная иерархия, высокие контрасты.
- Учитывай профессиональный tone: спокойно, надежно, без визуального шума.
- Визуально ориентируйся на стилистику АЧПП-каталога: аккуратные границы, мягкие тени, брендовые акценты.

### Вывод
1. Информационная архитектура мобильного приложения.  
2. User flows (входящие -> чат -> ответ -> закрытие).  
3. Screen-by-screen UI spec.  
4. Flutter-oriented component mapping (Scaffold, SliverAppBar, ListView, BottomSheet, FAB/CTA, etc.).  
5. Design tokens и рекомендации для адаптивности.

## 4) Промпт для Mobile Capacitor + Vue + Tailwind + DaisyUI

Сгенерируй **mobile UI-дизайн и компонентную систему** для Pulse под стек:
**Capacitor + Vue 3 + TailwindCSS + DaisyUI**.

Контекст проекта:
- Единый inbox для чатов из Telegram/VK/Web.
- Основной пользователь: модератор Портала АЧПП.
- Нужны инструменты AI-помощи для составления ответа и предложения решения.

### Цель дизайна
Создать мобильный интерфейс, который:
- быстро загружается и легко реализуется на Vue + Tailwind + DaisyUI,
- визуально спокойный, профессиональный, эмпатичный,
- удобен для ежедневной интенсивной модерации.
- использует бренд-палитру из общего контекста и dark-тему на базе `zinc` (не слишком темную).

### Что включить в дизайн
- Mobile shell + безопасные отступы (safe area), bottom nav/tab bar.
- Экран Inbox:
  - сегментированные табы (`Мои/Нераспределенные/Все`),
  - поисковая строка,
  - фильтр статуса (`Открытые/Закрытые`) как chips/dropdown.
- Экран Chat thread:
  - header с действиями (`Назначить`, `Закрыть`),
  - сообщения + вложения,
  - фиксированный composer внизу,
  - быстрые шаблоны ответов (chips/quick insert).
- AI panel (drawer или bottom sheet):
  - summary,
  - suggested replies,
  - recommended action.
- Settings экран (уведомления, тема, звук, профиль).

### Техническая ориентация (обязательно)
- Предложи структуру UI в терминах Vue компонентов:
  - `AppShell`, `InboxList`, `ChatCard`, `MessageBubble`, `ComposerBar`, `AiAssistPanel`.
- Предложи базу DaisyUI-компонентов: `navbar`, `tabs`, `input`, `textarea`, `badge`, `drawer`, `modal`, `toast`, `button`.
- Предложи Tailwind utility-паттерны для spacing/type/rounded/shadows.
- Учти PWA/Capacitor нюансы: keyboard overlap, safe-area insets, pull-to-refresh.
- Для цветов предложи Tailwind map для brand-палитры и `zinc` dark-surface scale (`zinc-900/950` + контрастные слои).

### Состояния
- Loading skeletons, empty list, no internet, request error, offline indicator.
- Четкие feedback-сценарии: отправка сообщения, загрузка файла, AI processing.

### Вывод
1. Mobile design direction.  
2. Набор экранов и иерархия.  
3. Компонентная библиотека (Vue + DaisyUI mapping).  
4. Tailwind token recommendations.  
5. UX rationale для скорости модерации и снижения ошибок.
